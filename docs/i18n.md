# i18n String Catalog

HostsFileGet now has a versioned UI string catalog foundation. The current app still ships in English, but new localizable text should move through the catalog instead of adding more hard-coded strings.

## Catalog Location

- Default catalog: `data/i18n/en-US.json`
- Schema version: `1`
- Runtime fallback: built-in English strings in `hosts_editor.py`
- In-app report: **Tools > Translation Catalog...**
- Template CLI: `python hosts_editor.py --i18n-template LOCALE OUTPUT.json`
- Validation CLI: `python hosts_editor.py --i18n-validate OUTPUT.json`

The launcher caches `data/i18n/en-US.json` beside the downloaded editor when possible. If the catalog is missing, the app falls back to the built-in English messages so cached script launches do not fail.

## Schema

```json
{
  "schema_version": 1,
  "locale": "en-US",
  "messages": {
    "common.close": "Close"
  }
}
```

Rules:

- Locale tags use a conservative BCP-47-style form such as `en-US`.
- Message keys use dotted lowercase namespaces such as `dialog.accessibility.title`.
- Values must be non-empty strings without control characters.
- Missing keys fall back to built-in English.

## Adding Strings

1. Add a stable key to `DEFAULT_I18N_MESSAGES` in `hosts_editor.py`.
2. Add the same key to `data/i18n/en-US.json`.
3. Use `self.t("namespace.key")` inside `HostsFileEditor` UI code.
4. Add or update tests for catalog validation or formatting when behavior changes.

Do not translate user data, hostnames, source names, paths, command output, or raw report payloads. Those values should remain exact.

## Translation Contributions

Generate a complete starting catalog for the target locale:

```powershell
python hosts_editor.py --i18n-template es-MX data/i18n/es-MX.json
```

Translate values under `messages` while preserving every key. Keep placeholders such as `{count}` intact; placeholder names are validated against the English fallback catalog.

Validate the contribution before opening a pull request:

```powershell
python hosts_editor.py --i18n-validate data/i18n/es-MX.json
```

Exit codes:

- `0`: catalog is structurally ready.
- `1`: catalog is valid JSON but needs work because keys or placeholders differ from the fallback catalog.
- `2`: catalog could not be read or failed schema validation.

The report also lists unchanged English strings for non-English locales. Those are review warnings, not hard failures, because product names and some technical labels may intentionally remain English.
